# 怎么借外部平台：按六层映射

核心原则先说清楚：**工具负责计算，project_objects.py 负责传递。**

每个工具的输出，都通过一个薄适配函数转换成标准对象，下一层只认标准对象，不认工具的原始输出。这样换工具时只改适配函数，其他都不动。

---

## 第一层：策略孵化

### 借什么

| 工具                  | 借的具体内容                          | 不借什么                   |
| ------------------- | ------------------------------- | ---------------------- |
| Claude / GPT        | 讨论策略逻辑、把模糊想法具体化为规则、检查规则里有没有前向偏差 | 不让它判断策略是否有 Edge，那是你的判断 |
| RD-Agent + Qlib（未来） | 自动生成因子假设、写测试代码、在 A 股数据上验证       | 不让它替代你对结论的审批           |

### 层边界输出

```
你（或 AI 辅助） → 填写 ResearchTask 对象
字段：策略名称、假设、品种池、入场规则、出场规则、过滤条件、为什么可能有 Edge
```

`ResearchTask` 是第一层唯一的输出物，进入第二层之前必须填完。

## 第二层：数据处理

### 借什么

| 工具      | 借的具体内容                                                             | 不借什么                      |
| ------- | ------------------------------------------------------------------ | ------------------------- |
| AkShare | `fund_etf_hist_sina()` 拉 ETF 日线，`stock_zh_a_hist()` 拉股票日线，历史退市股票接口 | 不借实时数据（不稳定），不借财务数据（暂时用不上） |
| Pandera | Schema 定义 + 数据进入时的漏斗校验                                             | 不借它做业务逻辑（哪些数据该丢、该保留是你的规则） |
| Parquet | 列式存储，读写效率高                                                         | 不借它做检索，索引由 system_db 管    |

### 连接方式

```
AkShare 返回 DataFrame
    ↓ market_data_v1.py（已有）
标准化列名 + 类型 + 后复权处理
    ↓ Pandera 校验（price > 0，无 NaN 等）
写入 Parquet 文件
```

**已完成，直接保留，不改。**

### 层边界输出

```
Parquet 文件（628 ETF + 1666 股票 + 退市股票补充后）
schema：date / open / high / low / close / volume / instrument_code
```

## 第三层：离线回测

这层是整个系统可信度的核心，工具最多，连接最关键。

### 借什么

| 工具         | 借的具体内容                                                                               | 不借什么                  |
| ---------- | ------------------------------------------------------------------------------------ | --------------------- |
| VectorBT   | `Portfolio.from_signals()` 跑向量化回测，速度快，用于初筛和参数搜索                                      | 不用它的最终指标数字，不信任它做最终结论  |
| Hikyuu     | 事件驱动引擎，原生 A 股，C++ 底层，内置涨跌停约束，用于交叉核验基线结论                                              | 不用它做参数搜索（太慢），不用它做日常实验 |
| QuantStats | `qs.stats.sharpe()` / `qs.stats.max_drawdown()` / `qs.reports.html()` 生成标准 tearsheet | 不用它做存储，不用它做策略逻辑       |

### 连接方式（最关键的一段）

```
读 Parquet → 生成信号 Series（0/1）
    ↓
VectorBT Portfolio.from_signals()
    ↓ 提取
portfolio.returns()  → pd.Series（日收益率序列）
portfolio.trades     → 每笔交易明细
    ↓
QuantStats 接收 pd.Series
    ↓ 输出
metrics dict：sharpe / max_drawdown / annual_return / win_rate / ...
    ↓ 适配函数（10行）
填入 ExperimentRun.metrics_summary（格式不变，来源变了）
    ↓
experiment_store.write(experiment_run)
```

Hikyuu 交叉验证走独立通道：

```
同一组参数 + 同一份数据
    ↓
Hikyuu 事件驱动回测
    ↓ 输出
hikyuu_sharpe / hikyuu_max_drawdown
    ↓ compare_results()
偏差 < 阈值？→ 结论可信 → 继续
偏差 > 阈值？→ 停下来找 bug
```

### 层边界输出

```
ExperimentRun 对象
含：metrics_summary（来自 QuantStats）/ rule_expression / dataset_snapshot
/ review_outcome / decision_status
→ 写入 experiment_store（JSON 文件）
→ 同步写入 MLflow
```

## 第四层：压力测试与优化

### 借什么

| 工具                | 借的具体内容                                                       | 不借什么               |
| ----------------- | ------------------------------------------------------------ | ------------------ |
| Optuna            | `create_study()` + TPE 采样器做参数搜索，`optuna.visualization` 生成热力图 | 不让它驱动 WFO（两件事分开）   |
| VectorBT Splitter | `vbt.Splitter.from_rolling()` 做时间序列滑动窗口切分，内置防数据泄露逻辑          | 不手写滑动窗口（容易出隐性 bug） |
| numpy             | 蒙特卡洛：打乱交易 P&L 列表 1000 次，重算最大回撤分布                             | 不引入新框架（逻辑简单，10行可控） |

### 连接方式

**Walk-Forward（借 VectorBT Splitter）：**

```
vbt.Splitter.from_rolling(index, length=252*3, offset=252)
    ↓ 生成多组 [train_idx, test_idx]
每组：train 上 Optuna 搜最优参数
      test 上 VectorBT 跑这组参数
    ↓
收集所有 test 期 Sharpe → WFO 稳定性得分
→ 写入 ValidationRecord
```

**蒙特卡洛（numpy）：**

```
从 ExperimentRun 取出 trades 列表（每笔 P&L）
    ↓
np.random.shuffle(trades) × 1000 次
每次计算 cumsum → max_drawdown
    ↓
统计：P(max_drawdown > 18%) = ?
→ 写入 ValidationRecord
```

**参数高原（Optuna 可视化）：**

```
Optuna study 跑完
    ↓
optuna.visualization.plot_contour(study, params=["entry_window","exit_window"])
    ↓
热力图：最优点在平缓区域 → 参数可信
        最优点是孤峰     → 直接废弃
```

### 层边界输出

```
ValidationRecord 对象集合
含：WFO 各窗口 Sharpe / 蒙特卡洛回撤分布 / 参数高原确认
→ 写入 validation_store（JSON 文件）
```

## 第五层：仿真交易

### 借什么

| 工具          | 借的具体内容                      | 不借什么                |
| ----------- | --------------------------- | ------------------- |
| 手动日志（第一阶段）  | Excel 或简单 JSON 记录每日信号和假设成交价 | 先不接任何系统             |
| VN.py（第二阶段） | 仿真账户接口，自动执行信号，中国券商协议适配      | 不借它写策略逻辑，策略逻辑永远在你这里 |

### 连接方式

```
每日收盘后，运行信号生成脚本
    ↓ 读取
最新 Parquet 数据（滚动更新）
    ↓ 执行
strategy.generate_signal(latest_data) → Buy / Sell / Hold
    ↓ 记录
paper_trade_log.json：{date, signal, assumed_price, actual_next_open}
```

VN.py 接入时，信号生成不变，只是执行层替换：

```
signal → VN.py order_manager → 仿真撮合 → 返回成交记录
```

### 层边界输出

```
纸交易日志（含实际 vs 假设滑点对比）
→ 进入 Layer 6 决策
```

## 第六层：实盘上线

### 借什么

| 工具          | 借的具体内容                    | 不借什么                  |
| ----------- | ------------------------- | --------------------- |
| VN.py 或 QMT | 券商 API 适配层，处理底层协议，撤单、补单逻辑 | 不借它的风控逻辑，熔断是你自己写的独立脚本 |

### 连接方式

```
策略配置（从 ExperimentRun 读取最终参数）
    ↓ 注入
VN.py 策略类（只有信号逻辑，10-20 行）
    ↓
VN.py order_manager → 券商柜台
    ↓ 同步
日志写入 + 每日净值更新
    ↓ 独立监控脚本
实盘回撤 > 18% → 触发熔断脚本 → 清仓 + 封禁
```

## 整体管道连通图

```
[草稿卡]                 ResearchTask
    ↓
[数据层]    AkShare → Pandera → Parquet
    ↓
[回测层]    Parquet → VectorBT → QuantStats → ExperimentRun
                           ↘ Hikyuu（交叉核验，独立通道）
    ↓
[优化层]    ExperimentRun → VectorBT Splitter(WFO) → ValidationRecord
                         → Optuna(参数高原)         → ValidationRecord
                         → numpy(蒙特卡洛)           → ValidationRecord
    ↓
[仿真层]    ValidationRecord → 信号脚本 → 纸交易日志
    ↓
[实盘层]    纸交易日志 → VN.py → 券商 → 熔断脚本守护
```

每个箭头都是一个明确的 Python 函数调用，没有魔法，没有隐式依赖。

---

## 这个组合是不是最好的配置

基于搜索结果和你的场景（Windows 11，A 股股票为主 + ETF，个人量化），结论是：

**是的，这是当前最务实的主流组合，没有明显更好的替代。**

唯一需要注意的两点：

- Hikyuu 比 Backtrader 更适合 A 股（Backtrader 开发已经基本停止）
- Qlib + RD-Agent 在你进入多因子股票研究时引入，现在不急
