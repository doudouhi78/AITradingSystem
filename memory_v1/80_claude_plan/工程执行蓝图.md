## Phase 2：补完 Layer 4 压力测试

**解决的交易问题：** exp-007 的 Edge 是真实的，还是对历史数据的记忆？

| 工程单    | 具体内容                                                       | 产出               |
| ------ | ---------------------------------------------------------- | ---------------- |
| WI-201 | 用 vbt.Splitter 实现 WFO：4个窗口（训练3年/测试1年，滑动1年）                 | wfo_runner.py    |
| WI-202 | 运行 exp-007 WFO，记录每个窗口的训练集 Sharpe 和测试集 Sharpe               | WFO 结果表          |
| WI-203 | 用 numpy 实现蒙特卡洛：从 exp-007 提取每笔交易 P&L，打乱 1000 次，重算最大回撤分布     | monte_carlo.py   |
| WI-204 | 运行蒙特卡洛，统计 P(最大回撤 > 18%)，画分布直方图                             | MC 结果图           |
| WI-205 | Optuna 跑 entry_window × exit_window 参数空间，生成等高线热力图          | 参数热力图            |
| WI-206 | 把 WFO + MC + 热力图结论整理进 ValidationRecord，写入 validation_store | ValidationRecord |

**验收标准：**

- WFO：测试集 Sharpe 均值 ÷ 训练集 Sharpe 均值 > 50%，且所有测试窗口 Sharpe > 0
- 蒙特卡洛：P(最大回撤 > 18%) < 20%
- 热力图：entry25/exit20 在平缓区域（相邻参数变化时 Sharpe 变化 < 15%）

**失败处理：**

```
WFO 测试集塌掉（Sharpe < 0 或 < 训练集 50%）
    → 不能继续，回 Layer 1 重审策略逻辑
    → 这不是 bug，是策略本身的问题，接受这个结论

蒙特卡洛破产概率过高
    → 回 Layer 3 调整仓位上限（进一步降低单次风险）
    → 重新跑 MC，直到通过

热力图是孤峰
    → 换参数区域，或换策略逻辑
    → 不能靠孤峰参数上实盘
```

## 每阶段决策规则

```
阶段验收通过 → 推进下一阶段

阶段验收失败
    ├── 属于工程 bug（代码错误）→ 修复，重新验收
    ├── 属于策略问题（过拟合/无 Edge）→ 回 Layer 1 重新出发
    └── 属于成本偏差（滑点/手续费）→ 回 Layer 3 修正参数
```

**不存在"将就通过"。** 验收标准是交易合理性的底线，不是形式检查。

## 三个问题的总结

问题 1 回答了**用什么工具、借什么、不借什么、怎么连通**。  
问题 2 回答了**每层解决什么交易问题、用什么交易方法、谁来负责**。  
问题 3 回答了**按什么顺序推进、做什么、怎么验收、失败怎么处理**。

三张图合在一起，就是你这个项目从现在到实盘的完整路线图。
