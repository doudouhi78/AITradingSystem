核心原则与 Phase 1 一致：**工具负责计算，project_objects.py 负责传递。Phase 2 不引入新的外部依赖，主要是移植老项目（D:\TradingSystem）的已有代码。**

## Module B：Gate 系统从老项目移植

### 老项目结构（来源）

```
D:\TradingSystem\ts\gate\
    drawdown_gate.py    → DrawdownGate 类：监控回撤，超阈值 → 阻断
    trend_gate.py       → TrendGate 类：大盘均线判断
    breadth_gate.py     → BreadthGate 类：市场宽度计算
    vol_gate.py         → VolGate 类：波动率分位数
    scheduler.py        → GateScheduler 类：统一调度，输出 allowed/blocked
```

### 移植目标（目的地）

```
D:\AITradingSystem\src\ai_dev_os\gate\
    __init__.py
    drawdown_gate.py    （适配版）
    trend_gate.py       （适配版）
    breadth_gate.py     （适配版）
    vol_gate.py         （适配版）
    gate_scheduler.py   （适配版，含 GateDecision TypedDict）
```

### 需要适配的内容

老项目的 Gate 读数据的方式和当前项目不同，需要替换数据接口：

| Gate           | 老项目读数据方式                | 新项目适配方式                              |
| -------------- | ----------------------- | ------------------------------------ |
| DrawdownGate   | 从老项目的 AccountManager 读净值 | 从 paper_trade_log.json 或 VBT 回测结果读净值 |
| TrendGate      | 从老项目数据库读指数数据            | 读 data/etfs/510300.sh.parquet         |
| BreadthGate    | 从老项目数据库批量读股票            | 遍历 data/stocks/ 下所有 Parquet，计算上涨面   |
| VolGate        | 从老项目数据库读指数               | 读 510300.sh.parquet 计算滚动波动率          |
| GateScheduler  | 老项目特有的调度协议              | 适配为接受 date、portfolio_value 两个参数      |

### 不借什么

- 不引入老项目的其他依赖（如老项目特有的 ORM、数据库连接等）
- 不把老项目整体迁移，只移植 gate/ 下的 5 个文件
- Gate 的触发阈值参数不从老项目照搬——老项目针对不同策略，参数需要重新审定

## Module C：仓位与止损从老项目移植

### 老项目结构（来源）

```
D:\TradingSystem\ts\risk\
    position_sizing.py
        class RBasedPositionSizer:
            R = equity × risk_per_trade × risk_multiplier
            qty = floor(R / (entry_price - stop_price))

D:\TradingSystem\ts\strategy\
    atr_stop.py
        def wilder_atr(high, low, close, period=14)
        def compute_entry_stop(entry_price, atr_value, multiplier=2.0)
```

### 移植目标（目的地）

```
D:\AITradingSystem\src\ai_dev_os\risk\
    __init__.py
    position_sizing.py  （适配版）
    atr_stop.py         （适配版）
```

### 需要适配的内容

- `RBasedPositionSizer` 的 `equity` 参数：在回测中来自 VectorBT portfolio 净值，在实盘中来自账户余额
- ATR 计算的数据来源：读当前项目的 Parquet 格式（old project 可能用不同列名）
- 新增仓位上下限约束（Phase 1 固定 50%，Phase 2 动态但需要 clip 到 [5%, 60%]）

### 连接到回测链路

```
读 Parquet → 计算 ATR(14)
                ↓
入场信号触发 → compute_entry_stop() → 止损价
                ↓
RBasedPositionSizer.compute_qty(equity, entry_price, stop_price) → 建议手数
                ↓
VectorBT 回测使用动态 position_fraction（而非固定值）
```

## Module D：仪表盘工具链

### 借什么

| 工具         | 具体使用方式                                                       | 为什么这个工具               |
| ---------- | ------------------------------------------------------------ | ---------------------- |
| QuantStats | `qs.reports.html(returns, output='report.html')`            | 已在项目中，tearsheet 标准格式   |
| Jinja2     | 自定义 HTML 模板，嵌入 Gate 状态、滑点对比等 QuantStats 没有的内容               | 轻量级，无数据库依赖，生成静态 HTML   |
| pandas     | 读 paper_trade_log.json 和 ValidationRecord，整合成报告数据           | 已在项目中                  |
| matplotlib | 在报告中生成 Gate 触发时间线图（可选，如果 QuantStats 不覆盖）                    | 已在项目中                  |

### 连接方式

```
paper_trade_log.json
    ↓ pandas 读取
实际滑点 vs 假设滑点对比表
    ↓

ValidationRecord（来自 validation_store）
    ↓
WFO ratio / MC 结果 / 含 Gate 后的结论
    ↓

Gate 状态日志（来自 GateScheduler 每日输出）
    ↓

以上 → Jinja2 模板 → strategy_report.html
同时 → QuantStats tearsheet → 嵌入或链接
```

### 不借什么

- 不用 Dash / Streamlit 做动态仪表盘（不需要服务器，静态 HTML 够用）
- 不引入 plotly 等新可视化库（matplotlib 已够用）
- 不做数据库持久化层（JSON 文件直接读取足够）
