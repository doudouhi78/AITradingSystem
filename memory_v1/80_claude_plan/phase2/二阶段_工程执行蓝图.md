## Module A：回测基础修正

**目标：** 修正滑点假设，确保所有后续模块的决策建立在可信的回测结论上。

**进入条件：** 无前置依赖，立即可执行。

| 工程单    | 具体内容                                                            | 产出                        |
| ------ | --------------------------------------------------------------- | ------------------------- |
| WI-A01 | 修改回测配置：将 slippage 从 0.05% 更新为 0.1%，同时更新 A 股往返成本表              | 更新后的回测配置文件                |
| WI-A02 | 重跑 exp-007 基线回测（VectorBT），记录新的 Sharpe / 最大回撤 / 年化收益            | 修正后的 ExperimentRun 记录      |
| WI-A03 | 重跑 WFO（6 窗口），更新 WFO ratio 和各窗口 Sharpe                          | 更新后的 ValidationRecord      |
| WI-A04 | 重跑蒙特卡洛（1000 次，position=50%），确认 P(max_dd>18%) 仍 < 20%            | 更新后的 MC 结果                 |
| WI-A05 | 对比修正前后结论，记录差异（Sharpe 差 / 回撤差 / WFO ratio 变化）                  | 修正对比文档                    |

**验收标准：**

- 滑点修正后 WFO ratio 仍 ≥ 0.5（如下跌至 0.3 以下，需上报停止推进）
- 蒙特卡洛 P(max_dd>18%) 仍 < 20%（否则需重新调整 position_fraction）
- 修正对比文档完整，新旧结论差异清晰

**失败处理：**

```
修正后 WFO ratio < 0.3
    → 停止 Phase 2，回报告用户：修正后策略统计显著性不足
    → 这不是工程问题，是策略本身在真实成本下无法成立

修正后 MC 破产概率超标
    → 回到 position_fraction，继续降低（如从 50% 降至 30%）
    → 重跑 MC 直到通过，再继续
```

```
    → 检查旧项目依赖，适配数据接口，不能引入旧项目的依赖包
```

| WI-C04 | 修改回测信号链路：在 VectorBT 回测中，将 position_fraction 替换为 R-based 计算出的动态仓位比例                              | 更新后的回测脚本                |
| WI-C05 | 重跑基线回测（exp-007，修正滑点，含动态仓位），对比与固定 50% 仓位的 Sharpe / 最大回撤 / 仓位分布差异                               | 动态仓位回测对比记录              |

**验收标准：**

- ATR(14) 计算结果与手工验算样本点误差 < 0.1%（验证 Wilder 平滑正确性）
- R-based 仓位下，单笔最大亏损 / 账户净值 ≤ 1.5%（允许少量超出，因为跳空）
- 动态仓位 MC 破产概率 < 20%
- signal_daily.py 每日输出包含 suggested_qty 字段

**失败处理：**

```
ATR 计算结果偏差 > 0.1%
    → 检查 Wilder 平滑初始化方式（第一个 ATR 值用 SMA 还是 TrueRange）
    → 修复后重新验证

动态仓位下 MC 破产概率超标
    → 降低 risk_per_trade（如从 1% 降至 0.5%）
    → 重跑 MC 直到通过
```

## 每阶段决策规则

```
工程单验收通过 → 推进下一个工程单

工程单验收失败
    ├── 工程 bug（代码错误）→ 修复，重新验收
    ├── 参数不合理      → 回到参数候选阶段，重新选参
    └── 策略根本性问题  → 上报用户，等待决策，不自行推进
```

**不存在"将就通过"。** Gate 系统和仓位逻辑直接影响真实资金，验收标准是底线。
