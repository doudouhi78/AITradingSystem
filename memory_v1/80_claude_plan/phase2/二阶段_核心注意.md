# 二阶段：核心注意

---

## 一、Gate 参数过拟合是最大风险

Phase 2 最容易犯的错误，不是代码写错，而是**Gate 参数调得太精准**。

Gate 系统有 4 个组件，每个有 1-3 个参数：DrawdownGate 的回撤阈值、TrendGate 的均线窗口、BreadthGate 的上涨面比例阈值、VolGate 的波动率分位数。如果你把这 8-10 个参数全部在历史数据上优化，你就在用 10 个自由度去"解释"历史，这和直接过拟合策略参数没有区别。

> **Gate 的目的是"在明显不好的市场环境阻断入场"，不是"找到历史上精确的入场窗口"。**

参数的合理设置方式：
- 基于常识和市场直觉先给定一个大致范围（如 TrendGate 用 120 日线，因为 120 日是半年线，业界通用）
- 验证在这个参数下 Gate 确实在历史大熊市中触发
- 不要去优化"触发几次、回避了多少回撤"——那是过拟合

---

## 二、Gate 加入后 WFO ratio 下降不一定是坏事

Phase 1 的 WFO ratio = 0.682。加入 Gate 后，如果 ratio 轻微下降（如到 0.55），不一定要删掉 Gate。

原因：Gate 减少了样本数（阻断了部分交易），样本数少时统计指标天然更不稳定。衡量 Gate 价值的更好问题是：

**Gate 触发期间，策略如果无视 Gate 强行入场，回撤会更大吗？**

这个问题比"WFO ratio 变化"更直接。用历史数据做个简单验证：在 Gate 阻断的区间强行回测，看净值曲线是否确实更差。

---

## 三、R-based 仓位会让回测结论和 Phase 1 无法直接比较

Phase 1 用的是固定 50% 仓位，所有验收标准（WFO ratio、MC 破产概率）都基于这个假设。

引入 R-based 动态仓位后，回测的年化收益、Sharpe、回撤会全部变化。**不要直接用 Phase 1 的阈值来判断 Phase 2 的结果是否通过。**

正确的做法：
- Module A 跑完（固定仓位+修正滑点）作为 Phase 2 的新基线
- Module C 跑完（动态仓位+修正滑点）对比这个新基线
- 只要 MC 破产概率 < 20%，WFO ratio ≥ 0.5，结论可接受

---

## 四、Gate 和熔断是两个完全不同的东西，不能混淆

**Gate**（Phase 2）：**事前**判断——市场环境不适合，不开新仓。这是交易逻辑的一部分。

**熔断**（Phase 6，circuit breaker）：**事后**兜底——策略已经亏损超出阈值，强制清仓。这是风控兜底，独立于策略逻辑。

两者都需要，不能用一个替代另一个。Gate 做不到"已经亏了就清仓"，熔断不能做"市场不好别入场"。

**Phase 2 只做 Gate，不提前做熔断。熔断的工程实现留到 Phase 6。**

---

## 五、ATR 止损会让部分历史交易无法执行

R-based 仓位的前提是有止损价格。如果止损价格和入场价格之间的距离极小（如 < 0.5%），计算出来的仓位会极大（接近全仓），需要设置仓位上限（如最大不超过账户的 60%）。

同时，高波动时 ATR 大，止损距离大，仓位会自动减小，这是设计好的行为，不是 bug。在验收时注意检查：

- 仓位计算结果的分布是否合理（不要出现 0.1% 或 200% 这种极端值）
- 设置仓位上下限约束（如 5% ≤ position_fraction ≤ 60%）

---

## 六、BreadthGate 数据需求最重，注意性能

BreadthGate 每天需要读取 1666 只股票的最新数据，计算"今日上涨股票比例"。这比读单只 ETF 慢很多。

注意：
- 不要在 signal_daily.py 中每次都重新读取所有股票的 Parquet
- 应该预计算每日市场宽度并缓存（每日更新一次，不需要每次运行都重算）
- 如果股票数据未更新（Parquet 是旧的），BreadthGate 会给出过期判断，这比没有 Gate 更危险

---

## 七、Phase 2 完成不等于可以上实盘

Phase 2 完成后，Gate 和动态仓位只是在历史数据上验证了。你需要：

1. 用新的完整系统重新跑 Phase 5 仿真（至少 4 周），观察 Gate 在真实市场环境下是否按预期触发
2. 确认 R-based 仓位计算在实际数据上没有极端值
3. 确认 BreadthGate 的数据管道稳定

**Phase 2 完成 → 重新进入 Phase 5 → Phase 5 通过 → 才能考虑 Phase 6。**

---

## 一句话总结

> Gate 系统不是"让回测更漂亮"的工具，是"在错误的市场环境中保护本金"的工具。参数不要调太精，阈值用常识判断，用历史熊市验证是否触发就够了。如果你在历史上优化 Gate 参数到极致，你只是做了一个更复杂的过拟合。
