# 六层的交易逻辑与实现对应关系

---

## 第一层：策略孵化

### 这一层解决的交易问题

> **你凭什么赚钱？你的 Edge 是什么？**

在进入任何验证之前，必须先回答这个问题。Edge 不是"感觉会涨"，是一个可以陈述的市场现象：趋势动量、均值回归、事件驱动、因子溢价。说不清楚 Edge 来源，后面全是空转。

### 具体交易方法

**第一步：裸逻辑 Edge 验证**  
假设每笔交易固定买入 1 万元，不计仓位、不加止损，只看：

```
胜率 × 平均盈利
─────────────── > 1.0（扣手续费后）
败率 × 平均亏损
```

这个值 ≤ 1.0，逻辑本身无效，直接放弃，不进第二步。

**第二步：加入生存止损**  
在裸逻辑基础上加最简单的硬止损（如跌破买入价 3%）。好的策略加入止损后，收益略降但回撤显著改善。如果加了止损反而更差，说明策略本身有问题。

**第三步：最简仓位雏形**  
引入 ATR 仓位（波动率倒数仓位）：每笔亏损对总资金的伤害恒定（如亏总资产 0.5%）。这不是最终仓位方案，是为了让后续比较有统一口径。

**关键原则：**

- 奥卡姆剃刀：参数越少越好，超过 4 个核心参数的策略大概率是过拟合
- 一个策略只解决一个问题，不要在一个策略里既抓趋势又做套利

| --------- | ----------------- | ----------------------- |
| Edge 来源定义 | **你自己**           | 这是 Alpha，AI 无法替代你对市场的认知 |

## 第二层：数据处理

### 这一层解决的交易问题

> **你的历史数据是否真实反映了那个时候你能知道的信息？**

量化里最隐蔽的死法不是策略差，而是数据有问题但你不知道。

### 具体交易方法

**后复权处理（A 股核心问题）**  
A 股的除权除息会在价格上产生跳空。必须用后复权计算涨跌幅，否则回测会把每次除权误判为暴跌。但下单时映射回未复权的真实价格。

**时间戳防火墙**  
T 时刻的信号，只能用 T-1 时刻及以前的数据。这听起来显然，但实际中很容易犯：

- 用当日收盘价决定当日买入（这是前向偏差）
- 财报使用财报所属季度日期而非公告日期

规范做法：T 日收盘后产生信号，T+1 日开盘成交。

**幸存者偏差消除**  
你现在的 1666 只股票都是当前活着的。如果回测只跑这些，等于提前知道"这些公司不会退市"，年化会虚高 5%-10%。必须把历史退市股票加入数据池。

**涨跌停处理（A 股特有）**  
涨停日成交量异常，价格不具参考性，不产生买入信号。跌停日流动性枯竭，无法正常卖出，必须在回测中模拟这个约束。

**Pandera 质检规则（业务层面）**

```
price > 0                    # 价格不能为负
volume > 0                   # 成交量不能为零
not 涨停日产生买入信号         # 涨停不能买
not 跌停日产生卖出信号         # 跌停不能卖
date 不重复                   # 不允许同一天两条记录
```

## 第三层：离线回测

### 这一层解决的交易问题

> **在真实交易成本和市场约束下，这个策略过去是否真的赚钱？**

注意：这一层的目标不是"让曲线好看"，而是"尽可能地证明它不赚钱"。能通过这一层的才算有价值。

### 具体交易方法

**成本注入（悲观假设）**

| 成本类型        | ETF         | A 股股票       |
| ----------- | ----------- | ----------- |
| 买入手续费       | 0.015%      | 0.03%       |
| 卖出手续费       | 0.015%      | 0.03%       |
| 印花税         | 无           | 卖出 0.1%     |
| 滑点          | 0.05%       | 0.05%-0.1%  |
| **单次交易总成本** | **约 0.08%** | **约 0.26%** |

股票的单次往返成本比 ETF 高 3 倍以上，这直接影响高频策略的可行性。

**市场约束**

- 涨停买不入：当日涨停，买入指令失败，跳过
- 跌停卖不出：当日跌停，卖出指令失败，持仓继续
- T+1 规则（股票）：当日买入，次日才能卖出
- 流动性约束：日均成交额太小的标的，跳过（防止冲击成本过高）

**信号时序规范**

```
T 日收盘：计算指标，生成信号（Buy/Sell/Hold）
T+1 日开盘：按信号执行，成交价 = T+1 开盘价
```

这是个人量化中最严谨的成交假设。用当日收盘价买入当日收盘信号，是隐性的前向偏差。

**基准比较**  
任何策略都必须和最简单的基线比，才能判断是否有真实改进：

- ETF 策略：对比买入持有该 ETF
- 股票策略：对比买入持有沪深 300

**向量化初筛 → 事件驱动核验**  
两步走，不是一步到位：

1. VectorBT 快速跑（向量化，秒级），先判断方向对不对
2. Hikyuu 核验（事件驱动，模拟真实撮合流程），确认结论可信

### 自研还是借用

| 内容         | 谁来完成                | 原因                         |
| ---------- | ------------------- | -------------------------- |
| 策略信号逻辑     | **你自己**             | 入场/出场规则是你的 Alpha           |
| 成本参数设定     | **你自己**             | 对应你实际使用的券商费率               |

## 第四层：压力测试与优化

### 这一层解决的交易问题

> **这个历史表现是真实 Edge，还是对历史数据的记忆？**

量化里最常见的死法：回测完美，实盘崩溃。根本原因是过拟合——策略"背"了历史答案，在未知的未来面前一塌糊涂。这一层的唯一目的是把过拟合的策略杀掉。

### 具体交易方法

**Walk-Forward Optimization（WFO）——最重要的一关**

把历史数据切成训练集和测试集，滚动前进：

```
2018-2020 训练 → 在 2021 测试
2019-2021 训练 → 在 2022 测试
2020-2022 训练 → 在 2023 测试
2021-2023 训练 → 在 2024 测试
```

每次用训练集找最优参数，直接在测试集（策略从未见过的数据）上执行。

**红线：** 测试集 Sharpe < 训练集 Sharpe 的 50%，判定过拟合，废弃。  
如果你的策略是真实 Edge，它在测试集上会和训练集接近，不会塌掉。

**参数高原检验**

找到最优参数后，把它周围的参数也跑一遍，画热力图：

```
                exit_window
           10    15    20    25
entry 20 [0.28  0.31  0.35  0.33]
entry 25 [0.30  0.35  0.38  0.36]  ← 最优区域
entry 30 [0.27  0.33  0.36  0.34]
entry 35 [0.24  0.29  0.31  0.30]
```

这叫"平缓山坡"，参数改变一点，收益变化很小。这种参数可以用。

如果最优参数是这样：

```
entry 24 [0.10  0.09  0.09  0.08]
entry 25 [0.09  0.08  0.38  0.08]  ← 孤峰！
entry 26 [0.09  0.10  0.08  0.09]
```

这叫"孤峰"，参数偏移一点性能崩溃，是典型过拟合，直接废弃。

**蒙特卡洛模拟**

把策略历史上每一笔交易的盈亏记录打乱顺序，重新组合 1000 次，模拟不同的交易路径。

核心问题：**如果运气不同，这个策略能不能活下去？**

```
原始顺序：[+2%, -1%, +3%, -2%, +4%, ...]
模拟路径1：[-2%, +4%, -1%, +3%, +2%, ...]
模拟路径2：[+3%, -2%, +4%, -1%, +2%, ...]
... × 1000
```

每条路径计算最大回撤，统计分布：

- 破产概率：P(最大回撤 > 你的承受极限)
- 如果 P > 20%，说明策略在"顺境中活得好，逆境中很危险"

**多目标优化原则**

不是"收益越高越好"，而是同时优化多个目标：

```
最大化：卡玛比率（年化收益 / 最大回撤）
约束：最大回撤 < 25%，交易次数 > 20（避免小样本）
```

## 第五层：仿真交易

### 这一层解决的交易问题

> **回测里的假设，和真实市场之间有多大的摩擦？**

这是从"实验室"到"战场"的过渡带。很多在第四层近乎完美的策略，到这里就因为执行延迟、成交偏差、系统问题崩溃。

### 具体交易方法

**全流程闭环测试**  
连续运行至少 4 周，不允许人工干预，模拟真实执行：

- 信号正确产生了吗？
- 记录的成交价和实际次日开盘价差多少？
- 系统在极端行情（大涨/大跌）时有没有异常？

**滑点审计（最关键）**  
每一笔假设成交都要记录：

```
回测假设成交价 vs 实际次日开盘价
```

如果实际滑点系统性地高于回测假设的 0.05%，必须停下来，回到第三层修正成本参数，重新回测。不修正就上实盘，等于用假数字做了决策。

**执行时序验证**  
确认你真的能在 T+1 开盘前做好准备：

- T 日收盘后何时运行信号脚本？
- 如果当晚有突发事件，次日开盘前你的判断会不会改变？
- 策略规则是否真的能机械执行，不受情绪干扰？

**异常处理规则**  
必须有明确的预案：

- 信号漏掉了怎么办（当天忘看）
- 网络断了怎么办
- 跌停卖不出怎么办（不是 bug，是策略本来就该这样）

### 自研还是借用

| ------- | ----------- | ----------------- |
| 信号生成脚本  | **你自己**     | 策略逻辑永远在你这里        |

## 第六层：实盘上线与风险监控

### 这一层解决的交易问题

> **在模型必然会失效的那一天，如何保住本金？**

这一层不是为了赚更多，是为了不死。所有量化策略都会失效，只是时间问题。这一层的核心是在失效发生时，你的账户还活着。

### 具体交易方法

**阶梯式部署（小资金试错）**  
第一阶段只用计划资金的 10%-20%，验证：

- 真实印花税、过户费对净值的侵蚀是否在预期内
- 真实滑点是否和仿真阶段接近

**策略哨兵（drift detection）**  
持续监控实盘表现是否偏离回测预期：

```
如果 实盘最大回撤 > 回测最大回撤 × 1.2
或   实盘净值曲线偏离回测分布 > 2 个标准差
→ 自动报警，人工复核，暂停新建仓位
```

**全局熔断（circuit breaker）**  
这段逻辑必须独立于策略之外，单独运行：

```
单日亏损 > 3%   → 当日停止交易
总回撤 > 18%    → 清仓 + 封禁下单权限，等待复核
```

熔断触发后，不是"先看看"，是立刻执行，事后再分析。

**执行纪律（量化最难的一关）**  
除非触发哨兵或熔断，严禁因为：

- 市场感觉不对
- 新闻让你害怕
- 最近连亏了几天

……而关掉程序或手动干预。这是量化和主观交易的本质区别。

**日志审计**  
每一笔实盘成交必须记录：

- 信号产生时间和依据
- 实际成交价 vs 假设价格
- 偏差原因（涨跌停 / 流动性不足 / 正常滑点）

这些记录不是给系统看的，是给你自己复盘用的。
