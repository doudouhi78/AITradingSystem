# Alpha研究三阶段：借助外部平台

核心原则同六层总原则：**工具负责计算，主对象层负责传递，研究结论永远由人审批。**

---

## Phase 1：打通基线管道（目标：因子→IC全链路跑通）

### 新引入工具

| 工具 | 借什么 | 不借什么 |
|---|---|---|
| **alphalens-reloaded** | `al.tears.create_full_tear_sheet()` 完整因子评估报告；IC序列、ICIR、IC衰减曲线、分组收益 | 不让它决定哪个因子好，那是你的判断 |
| **pandas_ta** | 批量计算技术指标（动量/ATR/均线），一行替代手写TA | 不用它的信号逻辑，只用纯计算 |
| AkShare（已有） | 日K数据（现有Parquet直接读）+ 基本面接口补充市净率/换手率 | 不引入新的行情源 |

### 不引入的工具（Phase 1明确排除）

- Qlib：Phase 2再引，Phase 1先用alphalens手动跑
- RD-Agent：Phase 3再考虑，现在先人工挖因子
- Barra模型：复杂度过高，Phase 1不需要

### Phase 1 平台连接链路

```
现有Parquet（close/volume/date/instrument）
    ↓ 读取 + 计算因子值
factor_df：MultiIndex(date, asset) → factor_value
    ↓
AkShare补充换手率/市净率（写入同一Parquet或单独factor_store）
    ↓
alphalens.utils.get_clean_factor_and_forward_returns()
    参数：factor=factor_df, prices=close_df, periods=(1,5,10,20)
    ↓
al.tears.create_full_tear_sheet(factor_data)
    输出：IC均值、ICIR、IC衰减图、分组收益图
    ↓
人工读报告 → 记录结论 → 写入factor_log.json
```

## Phase 2：系统化因子工程（目标：建立因子库，IC筛选流水线）

### 新引入工具

| 工具 | 借什么 | 不借什么 |
|---|---|---|
| **Qlib** | `qlib.data.D.features()` 批量获取A股因子数据（Alpha158因子集）；`SingleAssetDataHandler` | 不让它管策略逻辑，只用数据管道部分 |
| **scipy.stats** | `spearmanr()` 计算截面Rank IC；相关性矩阵降重复因子 | 不引入statsmodels（功能重叠） |
| **plotly** | 因子IC时间序列可视化、因子相关性热力图 | 不用matplotlib（交互性差） |
| **joblib** | 并行计算多个因子的IC，批量跑不阻塞 | |

### Phase 2 平台连接链路

```
因子候选池（自研 + Qlib Alpha158）
    ↓ 批量计算（joblib并行）
factor_store/ 目录：每个因子一个Parquet文件
    ↓ IC批量评估
ic_screening_pipeline.py：
    for each factor:
        alphalens → IC均值、ICIR、IC衰减半衰期
    → 筛选条件：|IC均值| > 0.03，ICIR > 0.5，衰减半衰期 > 5日
    ↓
通过筛选的因子列表 → factor_registry.json
    ↓ 相关性去重
scipy：因子间相关矩阵 → 相关性 > 0.7 的因子对保留ICIR更高的一个
    ↓
最终入库因子：有效因子库（约20-40个）
```

### Phase 2 新增外部数据源

| 数据 | 来源 | 用途 |
|---|---|---|
| 换手率 | AkShare `stock_zh_a_hist()` | 流动性类因子 |
| 市净率/PE | AkShare `stock_financial_analysis_indicator()` | 基本面类因子 |
| 北向资金净买入 | AkShare `stock_hsgt_north_money_em()` | 资金流类因子 |
| 融资融券余额 | AkShare `stock_margin_underlying_info_szse()` | 杠杆情绪类因子 |

## Phase 3：信号集成（目标：因子信号接入现有Gate+策略体系）

### 新引入工具

| 工具 | 借什么 | 不借什么 |
|---|---|---|
| **pyfolio + empyrical** | 绩效归因：策略收益分解，Barra-style因子暴露分析 | 不用pyfolio的仿真功能（已有VBT） |
| **optuna（已有）** | 因子权重优化：用贝叶斯搜索最优组合权重 | |
| **VectorBT（已有）** | 组合因子信号的历史回测验证 | |

### Phase 3 平台连接链路

```
有效因子库（factor_registry.json）
    ↓ 因子权重组合（Optuna搜索）
composite_signal = Σ(weight_i × factor_i_rank)
    ↓
信号生成：top分位 → 买入候选，bottom分位 → 卖出候选
    ↓
接入现有Gate系统（DrawdownGate/TrendGate等）
    Gate允许 → 候选信号生效
    Gate阻断 → 信号挂起
    ↓
VectorBT回测验证（接入现有ExperimentRun对象）
    ↓
pyfolio归因：哪些因子贡献收益，哪些是噪声
    ↓
写入ExperimentRun → 进入候选方案档案
```

## 三阶段工具引入节奏总览

```
Phase 1（第1-2周）：alphalens-reloaded + pandas_ta
Phase 2（第3-8周）：Qlib数据管道 + scipy + plotly + joblib
Phase 3（第9-12周）：pyfolio + empyrical（+ Optuna已有 + VBT已有）
```

**原则：不超前引入，每个工具的引入都必须有明确的当前任务驱动。**
