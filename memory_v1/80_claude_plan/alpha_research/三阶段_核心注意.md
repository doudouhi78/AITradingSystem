这里记录的不是方法论，而是因子研究中最常见、最致命的陷阱。每个坑都有人用真金白银踩过。

## 一、前向偏差（Look-Ahead Bias）——因子研究的头号杀手

**什么是前向偏差**

T日计算因子时，用到了T日之后才能知道的信息。

**A股因子研究中最常见的前向偏差形式**

| 偏差类型 | 错误做法 | 正确做法 |
|---|---|---|
| 财报数据 | 用Q1财报数据的"报告期"（3月31日）作为信号日期 | 必须用实际公告日（约4月20日前后），因为你3月31日看不到Q1财报 |
| 换手率/资金流 | 用T日收盘后才能确认的数据生成T日信号 | T日数据只能在T+1日盘前使用 |
| 指数成分 | 用当前的沪深300成分做历史回测 | 必须用历史时点的指数成分，不能用现在的 |
| 退市股票 | 回测样本只包含现存的股票 | 历史时点必须包含当时存在但后来退市的股票（幸存者偏差） |
| alphalens的prices参数 | 传入的forward_returns和factor_values不对齐 | factor在T日计算，forward_return从T+1日开盘计算起 |

**在alphalens中检验是否有前向偏差**

```python
# 正确的时间对齐方式
# factor_data的index应该是"因子确认可知的时刻"，不是"数据所描述的时刻"
factor_df.index  # 应该是 (T日收盘后确认的日期, 标的代码)
prices_df.index  # 应该从T+1日开始（实际可成交的价格）

# alphalens会自动对齐，但前提是你传入的prices本身没有前向偏差
```

**黄金规则：** 问自己——"在这个时间点，我真的能知道这个数字吗？"

## 二、多重检验问题（Multiple Testing Problem）——A股特有的放大版

**什么是多重检验问题**

测试100个因子，随机期望有5个看起来显著（p<0.05），但其实是噪声。因子数量越多，"偶然显著"的概率越高。

**A股市场的额外放大因素**

A股历史数据只有约30年，有效样本年数少；加上机构数量多、策略同质化，许多因子一旦被发现就迅速失效。这意味着A股的多重检验问题比成熟市场更严重。

**应对措施**

| 措施 | 具体做法 |
|---|---|
| 提高显著性门槛 | IC显著性要求：|IC均值| > 0.03，而不是>0.01 |
| Bonferroni校正 | 测试N个因子，p值门槛除以N（若测试50个因子，需p<0.001） |
| 独立样本验证 | 因子在训练期有效，必须在独立测试期再次验证才算通过 |
| 时间稳定性要求 | 分年度IC必须在大多数年份为正，不能靠某一年救全场 |
| 因子假说优先 | 先有交易逻辑假说，再跑数据验证——而不是先跑数据再找解释 |

**最危险的操作模式（明确禁止）**

```
❌ 批量跑500个因子 → 选IC最高的10个 → 就说"我发现了有效因子"
✅ 先提出20个有逻辑支撑的假说 → 跑IC → 通过的才算发现 → 独立样本复验
```

## 三、A股市场特殊性——不能直接照搬海外因子研究

**A股与成熟市场的关键差异**

| 维度 | A股特点 | 对因子研究的影响 |
|---|---|---|
| 涨跌停制度 | 10%涨跌停，ST股5% | 动量因子在涨跌停时无法成交，必须在回测中过滤 |
| T+1规则 | 当日买入次日才能卖 | 短周期动量因子（1日反转）需特别处理 |
| 机构同质化 | 大量量化策略策略相似 | 因子拥挤效应严重，发现到失效的周期比美股短得多 |
| 政策干预频繁 | 熔断/涨跌停调整/减持新规等 | 历史因子有效性受体制变量干扰，需分段验证 |
| 信息不对称 | 内部信息泄露仍普遍 | 基本面类因子对机构更有用，对个人投资者信息获取有限 |
| 风格轮动快 | 大盘/小盘、价值/成长轮动剧烈 | 单因子在某风格期极好/极差，ICIR不稳定 |

**A股因子研究的正确姿势**

1. **分市场状态检验**：牛市/熊市/震荡市中因子IC分别是什么，一个在全程都有效的因子才真正可用
2. **分时间段验证**：2016年前 vs 2016-2018量化元年 vs 2019至今，量化垄断化后因子衰减加速
3. **成交量和流动性过滤**：流动性差的标的任何因子都不可信，必须先设流动性门槛
4. **涨跌停过滤**：涨停日不产生买入信号，跌停日不产生卖出信号

## 四、因子容量问题——个人层面不是问题，但要理解

**机构担心的因子容量**

机构资金量大，买入动作本身会推动价格，挤出自己的Alpha。个人资金量小，容量不是主要问题。

**但有一个变体需要注意**

A股小市值股票往往流动性极差（日均成交额<500万），即使个人资金量小也可能造成明显冲击。**必须设置最低流动性筛选条件**：

```python
# 建议的最低流动性门槛（可根据实际资金量调整）
min_avg_daily_volume = 50_000_000  # 日均成交额 > 5000万
universe = universe[universe['avg_volume_20d'] > min_avg_daily_volume]
```

## 五、IC不等于可交易信号——从研究到执行的鸿沟

**IC高不等于策略好**

| IC情况 | 可能的原因 | 是否可交易 |
|---|---|---|
| IC高但衰减极快（<3日） | 日内Alpha，交易成本吃完 | 否（除非做高频） |
| IC高但只在某一年有效 | 偶发因素或某次政策周期 | 否 |
| IC高但top分位和bottom分位不单调 | 非线性关系，信号模糊 | 谨慎 |
| IC稳定，衰减半衰期>10日 | 中低频因子，个人可用 | 是 |

**从IC到可交易策略还需要**

1. 持仓周期设计（与IC衰减半衰期匹配）
2. 选股数量设计（top 5%还是top 20%？组合集中度）
3. 再平衡频率（日频/周频/月频）
4. 交易成本计算（换手率 × 单次成本 × 年化次数）
5. Gate过滤（市场不好时停止选股）

## 六、研究管理注意——防止研究资产流失

**研究腐坏的常见场景**

- 跑了100个因子，只记住了"有几个看起来不错"，细节忘了
- 3个月后想重新跑某个因子，但参数配置找不到了
- 同一个假说被测试了两次，第二次否定了第一次的结论，但两次结果都没留档

**解决方案：每个因子研究必须有留档**

```json
{
  "factor_id": "唯一ID",
  "hypothesis": "缝隙假说文字表述",
  "formula": "计算公式（代码级精确）",
  "data_requirements": ["需要哪些原始数据"],
  "test_date": "测试日期",
  "train_period": "用于IC测试的训练时间段",
  "ic_results": {},
  "conclusion": "有效/无效/存疑",
  "reason": "结论依据",
  "next_action": "入库/放弃/需补充数据再测"
}
```

**不要只记录"好"的因子**，无效因子同样是宝贵的研究资产——它告诉你哪条路走不通。

## 七、Phase 1 特定注意事项

1. **先验证数据管道，再解读IC值**：第一批因子跑出来的IC数字不重要，确认整个链路（数据→因子→alphalens→输出）无误才重要
2. **alphalens的时间对齐bug**：alphalens-reloaded比原版更新，但仍需手动确认forward_returns的时间点是T+1开盘，不是T日收盘
3. **中文标的代码处理**：AkShare返回的代码格式（600000.SH）和alphalens期望格式可能不同，需统一处理
4. **小样本警告**：如果只跑50只股票测3个因子，IC的统计显著性很低，Phase 1是为了验证工具链，不是下结论
