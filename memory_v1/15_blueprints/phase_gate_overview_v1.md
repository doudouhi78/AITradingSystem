## 三、第一阶段收口判定

第一阶段采用以下 5 条标准作为正式收口依据：

1. 最小闭环可重复跑通
2. agent 工位边界基本稳定
3. 研究对象和提交物模板已成型
4. 至少一个题材完成“基线 + 变体 + 复审”闭环
5. 用户能看懂并判断结果是否值得继续

当前判断：
- 上述 5 条已经全部满足
- 第一阶段的关键目标已经被证明：系统不是纸面结构，而是能从研究题目长出规则、最小验证和复审结果

## 四、当前阶段焦点判断

### 当前所在阶段
当前项目处于：
- 第一阶段：已完成并收口
- 第二阶段：玻璃盒研究系统建设，进行中

### 当前最重要的事情
当前最重要的不是扩展题材，也不是继续补文档，而是：
- 让研究链每个关键环节都变成可见工件
- 把风险与仓位模板开始纳入研究链
- 固定最小验证方法包
- 固化测试留存与实验索引路径
- 围绕当前临时基线继续做可解释的变体比较

## 五、阶段切换原则

- 未满足退出条件，不切下一阶段
- 如果某阶段文档存在但关键主干仍未形成，则视为未完成
- 下一阶段只能建立在上一阶段真实产物之上，不能靠口头默认进入
- 一旦进入新阶段，应从本表抽取当前阶段判断，写入 `10_mainline`
