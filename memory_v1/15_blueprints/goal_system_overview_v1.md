# AI交易系统目标系统总说明 v1

## 一、文档定位

这是一份上位总纲文档，用来把项目的外层目标、内层执行链、工位体系、工具体系和分阶段生长路线一次性串清。

它不替代：
- 项目总蓝图
- 阶段蓝图
- 当前主线
- 对象模板与制度文档

它的作用是作为上位索引，回答：
- 我们到底在建什么系统
- 这套系统完整长什么样
- 当前到底在补哪条主干

## 二、外层目标层

这个项目的外层目标是：

> 最终可稳定盈利的候选交易方案生产系统。

它不是量化平台本身，不是单条策略脚本集合，也不是聊天式研究助手。

它要持续产出、验证、复审、筛选和升级候选交易方案，并最终服务稳定盈利这一长期目标。

## 三、内层执行链总装图

为了逼近外层目标，系统内部需要建设一条完整但分层生长的执行链：

1. 市场认知与机会地图
2. 研究题目与假设生成
3. 数据层
4. 信号与策略表达层
5. 仓位与风险控制层
6. 回测与验证层
7. 组合与模拟层
8. 审批输出层
9. 复盘与反馈层

当前边界仍到：

> 研究 -> 组合 -> 模拟执行 -> 复盘反馈

不把真实实盘执行系统作为当前必建核心。

## 四、能力域总图

同一套系统从能力域看，应包含：
- 思想与方法域
- 任务与对象域
- 数据域
- 研究表达域
- 风险与仓位域
- 验证域
- 组合与模拟域
- 审批与解释域
- 经验与治理域

## 五、工位与角色层

### 1. 人的位置
- 你：方向裁决者、最终审批者、最终执行者

### 2. 建设者的位置
- 建设者：负责搭建、校正、修系统、升级系统

### 3. 运行态工位
- 真实主脑：编排与收敛
- 研究执行：研究与实验
- 复审审计：挑错与裁决

## 六、工具层

工具层只服务内层执行链，不主导项目目标。

当前关键工具包括：
- 数据工具：AkShare
- 回测工具：VectorBT
- 实验留存：runtime/experiments
- 实验索引：SQLite system_facts
- 后续图表与工件工具：逐步纳入

## 七、分阶段生长路线

当前采用 5 个高层阶段：
1. 最小研究闭环点火
2. 玻璃盒研究系统建设
3. 策略池 / 风险层建设
4. 组合与模拟建设
5. 接近真实执行

## 八、当前项目映射

当前项目状态：
- 第一阶段已完成并收口
- 当前处于第二阶段：玻璃盒研究系统建设阶段

当前补的主干不是扩模块数量，而是：
- 研究工件链显式化
- 风险与仓位模板进入研究链
- 最小验证包标准化
- 测试留存与实验索引稳定化
- 当前临时基线与变体比较机制清晰化
