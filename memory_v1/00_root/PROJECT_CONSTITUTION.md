# 项目宪法 v4

## 1. 根目标
本项目的根目标只有一个：

> 整合市场、前人经验、AI 生成能力、成熟工具、成熟验证体系，持续生产可供最终审批并可进入执行判断的完整候选交易方案，并最终服务稳定盈利目标。

## 2. 最高产物
本项目的最高产物只有一个：

> 完整候选交易方案。

不是想法，不是报告，不是页面，不是回测截图，不是系统状态。

## 3. 实现原则
项目实现只服从以下原则：
- 默认以资源整合为第一方法，不以自我发明为第一方法
- 先整合，后自建
- 先跑通流水线，后做复杂平台
- 先服务候选方案生产，后做辅助系统
- 工具、平台、页面都不得反向主导交易问题
- 项目进展优先看流水线是否更完整，不优先看系统是否更复杂

凡市场上已有以下成熟资源可用，必须优先整合进入主链：
- 市场机会与公开经验
- 前人方法与成熟样本
- AI 的生成与执行能力
- agent 的自动化能力
- 成熟工具
- 成熟验证体系
- 可复用的平台能力

只有当整合现有资源仍不能满足主链目标时，才允许自建。

## 4. 交易执行底盘
本项目最低限度必须持续补齐以下交易主干：

1. 市场与机会认知
2. 策略思想与规则表达
3. 数据与样本基础
4. 验证与比较机制
5. 风险与仓位管理
6. 执行约束
7. 复盘与反馈回流
8. 决策与纪律机制

任何建设动作都必须回答：它在补哪一条交易主干。

## 5. 人机职责关系
系统负责承接大部分研究、验证、比较、复审劳动。
人保留最终审核权、采用权、执行权。

人的长期位置只有四个：
- 研究任务发起者
- 学习者
- 最终审批者
- 现实执行者
