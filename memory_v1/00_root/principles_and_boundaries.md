# 项目总原则与长期边界 v3

## 一、总原则

1. 项目始终服务“完整候选交易方案”的生产，不服务系统自我膨胀。
2. 默认以资源整合为第一方法，不以自我发明为第一方法。
3. 主对象层必须自持；外部工具只能承接能力，不能成为真相源。
4. 验证体系优先借成熟方法与成熟引擎，不再默认扩自定义验证逻辑。
5. 工程推进统一按 6 层路线推进，不跳步，不混层。
6. 当前阶段优先解决研究闭环真实性与验证可信度，不优先解决前台展示。

## 二、长期边界

1. 不把项目定义成单一平台。
2. 不把项目定义成全自动代替人决策的系统。
3. 不把页面、图表、驾驶舱当成主目标。
4. 不把实验性验证组织层误当成成熟验证体系。
5. 不把外部平台对象直接暴露为项目主对象。

## 三、当前路线约束

当前统一采用的组合路线是：
- AkShare + Pandas + Pandera
- VectorBT
- Qlib + Optuna
- VN.py 或 QMT

其中：
- 自己掌握：研究对象、研究案卷、复审、决策状态
- 外部承接：数据、回测、优化、仿真、执行后段
