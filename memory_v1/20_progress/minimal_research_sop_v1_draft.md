# 最小研究 SOP v1 草稿

## 一、文件定位

这是一份用于规范系统最小研究链路的标准作业流程草稿。

它的目标不是描述所有未来可能流程，而是先定义第一版必须稳定跑通的主链：

研究任务单
-> 候选草案
-> 规则化
-> 快速验证
-> 完整补全
-> 二次验证
-> 候选交易方案卡
-> 审批提交包

## 二、适用范围

本 SOP 当前适用于：
- 阶段3制度设计
- 阶段4角色工作系统定形
- 阶段5最小研究闭环落地

它优先服务第一版最小闭环，不追求覆盖复杂分支。

## 三、最小研究流程

### 第一步：接收研究任务单

主脑先接收并确认研究任务单。

本步必须确认：
- 研究对象明确
- 研究主题明确
- 目标产物明确
- 个人约束明确
- 验证要求明确
- 当前阶段要求明确

如果这些内容不清，不能直接推进后续研究。

### 第二步：产出候选草案

研究执行工位先产出候选草案，不直接进入深度验证。

草案至少应覆盖：
- 准备抓取的市场现象
- 初步交易逻辑
- 初步仓位思路
- 初步风控思路
- 为什么值得继续

这一阶段的目的，是先判断方向是否值得往下走。

### 第三步：草案关停靠

主脑整理草案简报，交由你做第一轮方向判断。

这一关你重点只看：
- 方向是否偏了
- 是否值得继续
- 是否需要打回重做

若方向明显不成立，可在这里直接终止。

### 第四步：规则化

草案通过后，研究执行工位把草案压成可验证对象。

规则化至少应明确：
- 标的范围
- 入场规则
- 出场规则
- 仓位规则
- 风控规则
- 执行规则
- 基本成本假设

没有规则化，不进入验证。

### 第五步：快速验证

研究执行工位先做第一轮快速验证。

本轮验证重点不是精细优化，而是粗筛：
- 有没有基本活性
- 回撤是否离谱
- 成本后是否还活
- 是否明显只靠少数阶段支撑

这一步的目标是筛掉明显不值得继续的方向。

### 第六步：粗验证关停靠

快速验证后，复审工位给出第一轮复核意见，主脑汇总后交由你做第二轮判断。

这一关你重点只看：
- 还有没有生命体征
- 是否值得进入完整补全
- 是否应停止或返修

### 第七步：完整补全

只有通过粗验证的方向，才进入完整补全。

这一阶段需要补齐：
- 更完整的规则闭合
- 更具体的仓位安排
- 更具体的风控安排
- 更现实的执行方式
- 与个人约束的适配说明

到这一步，结果应开始接近正式方案卡。

### 第八步：二次验证

对完整补全后的方案做第二轮验证。

本轮重点是：
- 进一步确认收益与回撤特征
- 检查成本后表现
- 看关键阶段分布
- 补基本稳健性判断

这一步的目标是让最终提交物不只是“看起来不错”，而是经过第二轮确认。

### 第九步：形成候选交易方案卡

研究执行工位把完整结果整理成候选交易方案卡。

方案卡应能够：
- 独立表达完整方案
- 让复审与审批直接使用
- 不依赖大量口头补充才能成立

### 第十步：形成审批提交包

主脑汇总方案卡、验证摘要、复审意见和建议结论，形成审批提交包。

审批提交包的目标是：
- 让你在统一格式下完成审批
- 不需要自己重新拼接上下文

### 第十一步：终审关停靠

你基于审批提交包做最终判断。

这一关重点判断：
- 活性
- 稳健
- 适配
- 是否进入淘汰、观察、模拟或候选执行

## 四、三个人工停靠点

### 停靠点1：草案关
看方向，不看细节完美度。

### 停靠点2：粗验证关
看是否还有生命体征，值不值得继续。

### 停靠点3：终审关
看完整方案是否值得进入下一层真实采用判断。

## 五、当前明确禁止的做法

- 跳过研究任务单直接开做
- 跳过草案阶段直接深度验证
- 跳过规则化直接回测
- 跳过快速验证直接补全方案
- 跳过复审直接提交最终审批
- 把原始过程材料直接扔给审批者

## 六、当前草稿判断

这版 SOP 当前解决的是“第一版最小研究链路怎么顺序运行”，还没有完全解决：
- 特殊任务分支如何处理
- 打回重做时如何回流
- 与角色工位职责卡的精细衔接
- 与模板版本规则的联动方式

这些内容留待后续阶段3继续细化。
