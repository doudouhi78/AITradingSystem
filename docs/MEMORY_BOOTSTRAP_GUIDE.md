# 记忆体系使用说明

## 一、为什么这个项目也需要母体记忆骨架
因为 agent 引擎解决的是“怎么运行”，
而记忆骨架解决的是“项目怎么持续积累自己”。

如果没有这套骨架，新项目很容易出现：
- 所有结论都散在聊天里
- 过程草稿和长期结论混在一起
- lessons 和样本没有固定位置
- 后续项目接力困难

## 二、最小使用方式
起步时只要做到：
1. `00_root/vision.md` 写清项目愿景
2. `10_mainline/project_mainline_view.md` 维护当前阶段主线
3. `20_progress/working_draft_board.md` 放当前阶段草稿
4. `40_experience_base/working_test_draft_board.md` 放测试与经验草稿

## 三、什么时候往上归档
当一个阶段完成后：
- 把 20_progress 里真正稳定的结论归入 10_mainline
- 把测试草稿里反复成立的经验提炼为 lessons / samples
- 清空草稿层，为下一阶段准备

## 四、和角色系统的关系
角色系统负责：
- 运行时行动
- 角色级工作记忆

母体记忆骨架负责：
- 项目级长期状态
- 阶段判断
- 问题历史
- 经验沉淀

两者不是重复关系，而是层级关系。
