# 项目启动说明

## 一、项目目的

这个项目用于启动一个新的 AI 交易系统，并复用已经验证过的 agent 发动机基础能力。

当前目标不是一次性把完整交易系统流程都设计完，而是先搭一个可持续展开的项目底座。

## 二、当前已经包含的内容

- `src/ai_dev_os/`
  - 从 LangGraphLab 复制过来的通用 agent 发动机代码
- `config/agents.json`
  - 两个 starter 角色的干净配置模板
- `memory/roles/trading_zone/`
  - 本项目专属的交易角色包
- `runtime/`
  - 工作区和角色记忆目录预留

## 三、当前项目形态

这是一个“可展开的项目母体”，不是已经定型的交易系统成品。

它具备：

- 可复用的 agent 引擎
- 两个新角色的起步包
- 基础记忆机制和目录结构

它暂时不具备：

- 针对交易系统定制好的完整流程图
- 已经打磨完成的多阶段编排
- 已经产品化的驾驶舱

## 四、建议的下一步

1. 明确这个交易系统的第一阶段目标
   - 例如：先定义一个最小可回测策略闭环
2. 用 `market_architect` 把元目标展开成第一阶段蓝图
3. 用 `strategy_operator` 承接一个清晰的执行型 brief
4. 在真实工作中再决定要不要增加新的角色或编排

## 五、为什么现在先不做重编排

因为当前最需要的是：

- 快速进入真实项目
- 让真实问题逼出真正需要的流程

而不是先复制一整套旧项目的治理结构，把新项目一开始就绑死。
